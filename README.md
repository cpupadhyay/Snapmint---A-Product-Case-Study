🧠 Snapmint Product Case Study

Why Users Drop Off at the Moment of Purchase

👤 Chandra Prakash Upadhyay
📍 Bengaluru, India

🚀 Overview

Snapmint is a BNPL platform for users without credit cards.
Despite strong demand, users fail to complete purchases.

This case study identifies why users drop off and how to fix it.

❗ Problem

Users reach the checkout stage but don’t convert.

Key reason → Lack of trust, not lack of intent

👥 Users
Rahul (19–25): First-time buyer, fears losing money
Priya (24–28): Needs easy returns, smooth UX
Suresh (30–38): Highly skeptical, needs assurance

👉 Common need: Trust before transaction

🔍 Root Causes
No product reviews
No warranty visibility
No customer support
No cart system
Limited EMI options
No order tracking
🧠 Insight

Built for transactions, not for trust

💡 Solutions

P1 (High Impact)

Verified reviews
In-app support (chat/call)

P2

Cart + flexible EMI
User profile
7-day returns

P3

Order tracking
Better filters
📊 Impact (Expected)
Conversion: 3–5% → 12–15%
MAU: +25%
Cart abandonment: ↓30%
🎯 Takeaway
BNPL growth depends on trust design, not just pricing.
